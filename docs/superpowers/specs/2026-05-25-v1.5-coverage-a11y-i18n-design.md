# v1.5 — Coverage push + a11y / i18n polish — Design

- **Дата:** 2026-05-25
- **Ветка:** `release/v1.5.0` (от `release/v1.4.0`, опубликован v1.4.0)
- **Статус:** approved (brainstorming) → ready for implementation plan
- **Тип релиза:** аддитивный, **zero breaking**. Дефолты `locale`/`dir` = `en-US`/`ltr`, консюмеры без `LocaleProvider` не меняются.

## Контекст

v1.4 завёл светлую тему (семантические токены + `ThemeProvider`/`useTheme`/`ThemeScript`), motion-пресеты (`fadeIn/slideUp/scaleIn/popIn` + `useMotionPreset`) и внутренний `src/motion/overlayVariants.ts` (консолидация Popover/Tooltip + DatePicker/DateRangePicker). Baseline — **346 тестов**. Осознанно отложено в v1.5: рендер-smoke хвоста, a11y focus-rings (base form-controls на `outline-none` без видимого кольца — пред-существующий пробел, одинаков в обеих темах), i18n дат, RTL, прогон addon-a11y, motion-хвост Drawer/Sheet, raw→semantic хвост.

Замеры (Explore 2026-05-25):
- Тест-инфра: happy-dom; `framer-motion` глобально замокан в `vitest.setup.ts` (passthrough — стрипает анимацию, **прокидывает `dir` в DOM** → ассертим логическое состояние). Нет глобальных моков `IntersectionObserver`/`ResizeObserver`/`canvas`/`matchMedia`.
- focus-visible: `Input` (`input.const.ts`), `Button` (`button.const.ts`), `SelectFakeInput` — `outline-none` без кольца. Прецедент в репо: `SegmentedControl`/`TagInput` (`focus-visible:ring-2 ring-pr_purple`).
- i18n: `calendar.ts` `WEEKDAYS=['Mo'..'Su']` (Monday-first) + `formatDisplayDate`/`formatMonthLabel` хардкодят `'en-US'`; `GanttTimeline` уже на `toLocaleDateString(undefined,…)`. Прецедент Intl: `src/utils/date.ts#formatAbsoluteTime(locale?)`.
- Провайдеры локали нет; `ThemeContext` — только тема. `SagtechUIProvider` композирует `ThemeProvider` через `useThemeWrapper`.
- RTL: `dir`/`ps-`/`rtl:` в коде отсутствуют (greenfield). Асимметрия в overlays (Drawer/Tooltip/Popover/Toaster/Modal/BottomSheet) и form-controls (Input label, Select chevron).
- `@storybook/addon-a11y` **уже подключён** (`.storybook/main.ts:11`) + в devDeps → остаётся прогон, не настройка.
- CI: `.github/workflows/publish.yml` — `checkout@v4`, `setup-node@v4`, `pnpm/action-setup@v4`, `node-version: 20`. Один workflow-файл. GitHub форсит Node 24 с 2026-06-02.

## Цели

1. Рендер-smoke для хвоста: VirtualList / RichTextEditor / VisualGraphEditor, 10 canvas-чартов, 4 3D-сцены.
2. Видимые theme-aware `:focus-visible` кольца на base form-controls (обе темы).
3. i18n дат через `Intl` + новый `LocaleProvider` (`{ locale, dir }`).
4. RTL для overlays + form-controls через Tailwind v4 logical properties.
5. Прогон `addon-a11y` по сторям + починка находок.
6. Опц. долги v1.4: motion-консолидация Drawer/Sheet, raw→semantic хвост (high-traffic), недостающие Storybook-стори.
7. CI: бамп `publish.yml` под Node 24.

## Non-goals

- Полная миграция всех компонентов на семантику (хвост вне high-traffic).
- RTL-зеркалирование геометрии GanttTimeline-оси (absolute-`left` математика).
- Визуальные снапшоты/Playwright (smoke-рендер достаточно; Playwright — опциональный backlog-путь).
- Кастомные темы/локали за пределами `dark|light` / BCP-47 строки.

## Принятые решения (развилки, согласовано с пользователем)

| # | Развилка | Решение |
|---|---|---|

## Track D — i18n дат (LocaleProvider + Intl)

### D.1 LocaleProvider (зеркало Theme-трио)
- `src/providers/LocaleContext.tsx` (RSC-safe, **без** `'use client'`): `type Direction='ltr'|'rtl'`; `LocaleContextValue={locale:string;dir:Direction}`; `DEFAULT_LOCALE='en-US'`, `DEFAULT_DIR='ltr'`; `LocaleContext` с этим дефолтом; `useLocale()`. **Дефолт контекста сохраняет текущее поведение** — любой консюмер без провайдера получает `en-US`/`ltr`.
- `src/providers/LocaleProvider.tsx` (`'use client'`): controlled-only, `locale?='en-US'`, `dir?='ltr'`, memoized value. Без storage/matchMedia/DOM → SSR-детерминирован, нет FOUC, нет аналога `ThemeScript`.
- Композиция в `SagtechUIProvider`: `locale?`/`dir?` пропы, оборачивает `themed` в `<LocaleProvider>` только когда задан хотя бы один (зеркало `useThemeWrapper`).
- Экспорт из `providers/index.ts` + `src/index.ts` (группа Providers): `LocaleProvider, LocaleContext, useLocale`; типы `LocaleProviderProps, Direction, LocaleContextValue`.

### D.2 Locale-aware хелперы (`calendar.ts`, internal — 2 консюмера)
- `getWeekdayLabels(locale='en-US'): string[]` — Monday-first через `Intl.DateTimeFormat(locale,{weekday:'short'})` по фикс-референсу понедельника (`Date.UTC(2024,0,1+i)`), memoized в `Map`. **Сетка остаётся Monday-first** — `getFirstDayOfMonth` (Sun→6) не трогаем.
- `formatDisplayDate(date, locale='en-US')`, `formatMonthLabel(year, month, locale='en-US')` — опциональный `locale` с дефолтом `'en-US'` (вывод идентичен текущему). `WEEKDAYS` оставляем (минимизируем диф).

### D.3 Проводка
`useLocale()` + `locale?`-проп override (`prop ?? context`, дефолт `en-US`): `DatePicker`, `DateRangePicker` (weekday/month/display), `GanttTimeline` (`undefined`→`locale` в 3 вызовах — **задокументированный сдвиг дефолта** browser→en-US для тиков, только лейблы).

## Track E — RTL (logical properties)

**Маппинг:** `pl-→ps-`, `pr-→pe-`, `ml-→ms-`, `mr-→me-`, `left-→start-`, `right-→end-`, `border-l→border-s`, `border-r→border-e`. Симметрия/центр (`left-1/2 -translate-x-1/2`, `left-0 right-0`, block-ось `top/bottom/items-*`) — без изменений.

**`dir`:** компонент читает `dir` из `useLocale()` и ставит `dir={dir}` на свой корень. **Порталы обязаны ставить `dir` на корень портального поддерева** (DatePicker popover, Modal, Toaster, BottomSheet). In-tree (DateRangePicker, Tooltip, Popover, Drawer) — `dir` на свой корень; листовые form-controls (Input, SelectFakeInput) наследуют от хоста.

**Точечно через `rtl:` (logical недостаточно):** CSS-треугольники стрелок Tooltip/Popover (`border-l/r` → `rtl:` swap сторон); `Drawer.slideVariants` x-знак инвертируется в рантайме при `dir==='rtl'` (из `useLocale()`).

**Задокументированные ограничения:** GanttTimeline-ось остаётся LTR (геометрия на `left:`-математике); `directionalFadeVariants` 4px x-офсет не зеркалится (незаметно на fade).

## Track G — Motion-консолидация Drawer/Sheet

После Track E: общий фактори `edgeSlideVariants(edge, dir)` в `overlayVariants.ts` — точная пружина (`damping:30, stiffness:300`) + ключи `hidden/visible/exit` + dir-знак. **`Modal` (CSS `modalAnim`) и `SelectDropdownLayout` (`onAnimationComplete`) НЕ трогаем** (риск регрессии > выгода). Overlay-тесты зелёные.

## Track H — raw→semantic хвост

Идентично в dark; настоящая семантика в light. Чарты — после Track B.

## Риски / mitigations

- **Monday-first сетка** → лейблы выводятся в Monday-first порядке от фикс-референса; `getFirstDayOfMonth` не трогаем; тест на `getWeekdayLabels('en-US')[0]`.
- **Portal dir inheritance** (центральный риск) → `dir` явно на корне каждого портала; portal-dir smoke-гард.
- **GanttTimeline default shift** → задокументировано (только тики).
- **`border-s/e` доступность** в Tailwind v4.2.2 → подтверждено; при сбое fallback на `rtl:` точечно.
- **346 зелёных** → ни один тест не ассертит мигрируемые физические классы; framer-mock прокидывает `dir`; прогон suite между фазами.
- **TS 6.0 quirk** (`ignoreDeprecations: "6.0"`) — не трогать; `pnpm build` после изменений.
