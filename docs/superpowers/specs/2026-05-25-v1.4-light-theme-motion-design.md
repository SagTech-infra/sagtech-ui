# v1.4 — Light theme + Motion presets — Design

- **Дата:** 2026-05-25
- **Ветка:** `release/v1.4.0` (от `release/v1.3.1`, опубликован v1.3.1)
- **Статус:** approved (brainstorming) → ready for implementation plan
- **Тип релиза:** аддитивный, **zero breaking**. Консюмеры без `ThemeProvider` остаются на dark.

## Контекст

`@sagtech-infra/ui` — dark-only design system на Tailwind v4 (`@theme`-блок в `src/tokens/theme.css`, нет `tailwind.config`). Токены-цвета (`--color-black_1..4`, `--color-white_1..4`, `--color-grey_*`, `pr_purple`/`sec_purple`/`pr_blue`/`sec_blue`, status) генерят утилиты (`bg-black_1`). v1.3 уже завёл motion-инфраструктуру: токены `--motion-duration-*` / `--motion-ease-*` и `src/utils/motion.ts` (`motionDurationS`, `motionEaseBezier`, `tokenTransition`).

Замеры по кодовой базе:
- ~117 компонент-`.tsx`, raw-классы (`bg-black_1`, `text-white_2`, `border-grey_3`) рассыпаны густо (DatePicker 25, Pagination 23, GanttTimeline 22, Combobox 21, …).
- Дублирование motion-вариантов: `dropdownVariants` ×3 (`SelectDropdownLayout`, `DatePicker`, `DateRangePicker`), `motionVariants` ×2 (`Popover`, `Tooltip`), `slideVariants` (`Drawer`), `sideToVariants` (`Sheet`).
- Порталы (Modal/Drawer/Toast/Popover/DropdownMenu/CommandPalette) рендерятся в `document.body`.
- Storybook `preview.ts` знает фон dark/light, но theme-switcher'а нет.

## Цели

1. Светлая тема, переключаемая `dark | light | system`, покрывающая фоны/текст/границы/поверхности/статусы.
2. Семантический слой токенов, на который пишутся новые/мигрируемые компоненты.
3. Мгновенное разумное покрытие всех 117 компонентов без правок (override raw-scale под light).
4. WCAG AA контраст в обеих темах, видимые focus-rings, smoke на overlays.
5. Storybook theme-switcher.
6. Motion-пресеты (`fadeIn`/`slideUp`/`scaleIn`/`popIn`) поверх токенов v1.3 + консолидация дублей.

## Non-goals

- Миграция всех 117 компонентов на семантику (хвост — через raw-override, мигрируется в v1.5+).
- Кастомные пользовательские темы за пределами dark/light (backlog `docs/THEMING.md`).
- Light-снапшоты/визуальная регрессия (smoke-рендер достаточно).

## Принятые решения (brainstorming)

| # | Развилка | Решение |
|---|---|---|
| 1 | Стратегия покрытия | **Гибрид**: семантический слой + override raw-scale под `[data-theme=light]`; точечная миграция high-traffic компонентов на семантику |
| 2 | Провайдер | **Отдельный `ThemeProvider`** + `useTheme()`; `SagtechUIProvider` опционально композирует |
| 3 | Применение темы | `data-theme` + `style.colorScheme` на **`<html>`** (`target?: 'html'｜'body'`, default `'html'`) — покрывает порталы |
| 4 | Motion API | **Чистые функции-пресеты** + `useMotionPreset` (reduced-motion живёт в хуке) |
| 5 | Light-палитра | **Вывожу AA-совместимую** (сохраняя бренд-hue), `check-contrast.mjs` верифицирует, ревью в PR |
| 6 | FOUC/persist | **Полный набор**: `<ThemeScript/>` + localStorage (`sagtech-ui-theme`) + system |

## Каскад (механика Tailwind v4)

Семантика регистрируется как `--color-bg-primary` и т.п. в `@theme` → генерит утилиты `bg-bg-primary` (компилируется в `background-color: var(--color-bg-primary)`). Поскольку утилита ссылается на CSS-переменную в рантайме, переопределение переменной под `[data-theme="light"]` каскадирует без изменения внутренностей утилиты. Под light переопределяем **и** семантику, **и** raw-scale.

Акценты (`pr_purple`/`sec_purple`/`pr_blue`/`sec_blue`) под light корректируются точечно только если проваливают контраст.

## Track B — `ThemeProvider` + инфраструктура

### B.1 Файлы

- `src/providers/ThemeProvider.tsx` (`'use client'`) — логика.
- `src/providers/ThemeScript.tsx` — blocking inline-скрипт.
- `src/providers/__tests__/ThemeProvider.test.tsx`.
- Обновить `src/providers/index.ts` и `src/index.ts` (группа Foundations).

### B.2 API

```ts
type Theme = 'dark' | 'light' | 'system';

interface ThemeProviderProps {
  theme?: Theme;                  // controlled
  defaultTheme?: Theme;           // uncontrolled, default 'dark'
  onThemeChange?: (t: Theme) => void;
  target?: 'html' | 'body';       // default 'html'
  storageKey?: string;            // default 'sagtech-ui-theme'
  enableSystem?: boolean;         // default true
  children: ReactNode;
}

function useTheme(): {
  theme: Theme;            // выбранное ('system' остаётся 'system')
  resolvedTheme: 'dark' | 'light';  // фактическое после matchMedia
  setTheme: (t: Theme) => void;
};
```

- Эффект: пишет `data-theme={resolvedTheme}` + `style.colorScheme` на target-элемент. `system` подписывается на `matchMedia('(prefers-color-scheme: dark)')` и обновляет `resolvedTheme`.
- Uncontrolled: persist в localStorage по `storageKey`. Controlled (`theme` задан): persist отключён, источник истины — проп.
- SSR-безопасность: чтение localStorage/`matchMedia` только в эффекте; стартовое значение из props.

### B.3 `<ThemeScript />`

Рендерит `<script dangerouslySetInnerHTML>` с минифицированным IIFE: читает localStorage[storageKey] (или `prefers-color-scheme` при `system`/отсутствии) и ставит `data-theme` + `colorScheme` на target **до** гидрации → нет FOUC. Props: `{ storageKey?, defaultTheme?, target? }`. Документируется для вставки в `<head>` (Next `app/layout`).

### B.4 `SagtechUIProvider`

Получает опциональные `theme`/`defaultTheme`/`themeTarget`. Если заданы — оборачивает children в `ThemeProvider`. Иначе — поведение как сейчас (back-compat). Логика темы не дублируется, только композиция.

### B.5 Storybook

- `preview.ts`: `globalTypes.theme` (toolbar dropdown dark/light) + decorator, выставляющий `data-theme` на `document.documentElement` iframe-превью и синхронизирующий `parameters.backgrounds`.
- Дефолт — dark (как сейчас).

## Track D — Motion presets

### D.1 Файлы

- `src/motion/__tests__/presets.test.tsx`.
- Экспорт из `src/index.ts` (новая группа Motion).

### D.2 API

```ts
interface MotionPresetOptions {
  distance?: number;   // px, для slideUp/popIn
  duration?: keyof typeof motionDurationS | number;  // default 'normal'
  ease?: keyof typeof motionEaseBezier;              // default 'standard'
}
// чистые функции:
function fadeIn(o?: MotionPresetOptions): Variants;
function slideUp(o?: MotionPresetOptions): Variants;
function scaleIn(o?: MotionPresetOptions): Variants;
function popIn(o?: MotionPresetOptions): Variants;

// хук — оборачивает useReducedMotion(): при reduce → opacity-only / instant
function useMotionPreset(
  name: 'fadeIn' | 'slideUp' | 'scaleIn' | 'popIn',
  options?: MotionPresetOptions,
): Variants;
```

Все варианты имеют ключи `hidden` / `visible` (+ `exit` где нужно для `AnimatePresence`). Дефолты тянутся из `motionDurationS`/`motionEaseBezier` (v1.3).

### D.3 Консолидация

Перевести на пресеты, удалив локальные дубли (визуально идентично):
- `Popover` + `Tooltip` → `scaleIn`/`fadeIn`.
- `SelectDropdownLayout` + `DatePicker` + `DateRangePicker` (3× `dropdownVariants`) → общий пресет.
- `Drawer` (`slideVariants`) + `Sheet` (`sideToVariants`) → `slideUp`/направленный slide.
- Существующие тесты overlays должны остаться зелёными (framer-motion замокан в `vitest.setup.ts` — ассерты на логическое состояние, не на анимацию).

## Риски / mitigations

- **Raw-override даёт неверный flip там, где цвет использован буквально** (напр. фиксированный чёрный логотип). → Точечная миграция high-traffic + ручной smoke в Storybook light; задокументировать в `THEMING.md`.
- **FOUC при SSR** → `<ThemeScript/>` + документированная вставка в `<head>`.
- **Порталы вне provider-поддерева** → `data-theme` на `<html>` (не на враппере) покрывает `document.body`.
- **Регрессия motion при консолидации** → пресеты воспроизводят текущие значения; прогон существующих overlay-тестов.
- **TS 6.0 quirk** (`ignoreDeprecations: "6.0"`) — не трогать tsconfig; проверить `pnpm build` после изменений.
